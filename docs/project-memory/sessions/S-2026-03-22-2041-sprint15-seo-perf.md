## Goal

Improve SEO, page performance, and web standards compliance for davidbmar.com.

## Context

Sprint 15 checkpoint focused on SEO and accessibility. Meta descriptions still referenced 42 repos (now 104). No structured data, sitemap, robots.txt, or reduced-motion support.

## Plan

1. Add JSON-LD structured data (WebSite + Person schemas)
2. Update meta/OG/Twitter descriptions to 104 repos
3. Add robots meta tag
4. Add crossorigin attributes to external scripts
5. Create sitemap.xml and robots.txt
6. Add prefers-reduced-motion and focus-visible styles to CSS

## Changes Made

- **web/index.html**: Added robots meta tag, updated all descriptions from 42 to 104 repos, added JSON-LD WebSite and Person structured data, added crossorigin attributes to external scripts
- **web/sitemap.xml**: New file with entries for /, /search, /clusters, /access
- **web/robots.txt**: New file with Allow all and sitemap reference
- **web/css/style.css**: Added focus-visible outline styles for keyboard navigation (WCAG 2.1 AA), added prefers-reduced-motion media query to disable animations

## Decisions Made

- Used `crossorigin="anonymous"` on Google GSI and D3 scripts. SRI hashes not added because Google GSI doesn't publish stable hashes and D3 CDN versions may vary.
- Used `focus-visible` (not `:focus`) to avoid showing outlines on mouse clicks while maintaining keyboard accessibility.
- Used `!important` in reduced-motion query following established best practice — it needs to override all animation declarations.

## Open Questions

- Sitemap uses hash-based routes (#/search) which crawlers may not follow. Consider server-side rendering or History API routing in a future sprint.
