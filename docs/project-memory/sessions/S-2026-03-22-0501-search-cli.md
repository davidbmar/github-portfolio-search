## Goal

Build the semantic search function (SearchEngine class) and Click-based CLI tool for the GitHub Portfolio Search project.

## Context

Sprint 1 multi-agent build. agentA owns project scaffold + GitHub API client, agentB owns embeddings + vector store, agentC owns search + CLI. Merge order: A -> B -> C. Other agents have not yet committed, so this branch codes against the interfaces described in the sprint brief.

## Plan

1. Create `src/ghps/search.py` with SearchEngine wrapping VectorStore + EmbeddingPipeline
2. Create `src/ghps/cli.py` with Click CLI (search + index commands)
3. Add `pyproject.toml` with `[project.scripts]` entry (will merge with agentA's version)
4. Create comprehensive tests that mock the dependencies from other agents

## Changes Made

- `src/ghps/search.py`: SearchEngine class with deduplication (best chunk per repo), cosine similarity ranking
- `src/ghps/cli.py`: Click CLI with `ghps search` and `ghps index` commands, pretty-printed output
- `pyproject.toml`: Project config with `ghps = "ghps.cli:main"` script entry
- `tests/test_search.py`: 11 tests covering SearchEngine logic + CLI output formatting
- `src/ghps/__init__.py`: Minimal package init (needed for imports on this branch)
- `tests/__init__.py`: Test package init

## Decisions Made

- **Lazy imports in CLI**: Heavy dependencies (sentence-transformers, sqlite-vec) are imported inside command functions, not at module level, to keep `ghps --help` fast.
- **Dedup strategy**: Fetch 3x top_k candidates from store, then keep only best-scoring chunk per repo. Ensures enough results survive dedup.
- **Mock strategy for tests**: Since agentB's modules don't exist on this branch, CLI tests inject fake modules into `sys.modules` to test the full Click command flow.
- **pyproject.toml overlap**: Created a full pyproject.toml knowing agentA also creates one. The merge will need to reconcile these — agentA's should be the base, with `[project.scripts]` added from this branch.

## Open Questions

- VectorStore.search() return format: assumed `list[dict]` with keys `repo_name`, `text`, `score`, `source`, `url` — needs to match agentB's implementation.
