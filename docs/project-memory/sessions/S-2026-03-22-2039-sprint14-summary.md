## Goal

Sprint 14 delivered automated reindexing via GitHub Actions, freshness tracking in the data pipeline, and freshness badge display in the web UI.

## Context

After Sprint 13 added Google OAuth and gated access, Sprint 14 focused on keeping indexed data fresh. The site had 104 repos but no automated way to re-index when repos changed. Users had no visibility into how stale the data was.

## Plan

Three agents worked in parallel:
- **agentA-freshness-ui**: Freshness badges and last-indexed timestamps in the web UI
- **agentB-reindex-actions**: GitHub Actions workflow (weekly cron + manual dispatch), graceful error handling when no index exists
- **agentC-data-pipeline**: `indexed_at` tracking in the store, `freshness` labels in search results, dry-run mode for indexer

## Changes Made

### agentA — Freshness UI
- Added freshness badge rendering in the web UI (today/this week/this month/stale)
- Last-indexed timestamp displayed on repo cards

### agentB — Reindex Actions
- `.github/workflows/reindex.yml` — GitHub Actions workflow with weekly cron and manual dispatch
- `scripts/reindex.sh` — standalone idempotent reindex script
- `src/ghps/api.py` — graceful error handling when no index exists (returns empty results + error message instead of 500)
- `tests/test_api_no_index.py` — 4 tests for graceful error handling

### agentC — Data Pipeline
- `src/ghps/store.py` — `indexed_at TEXT` column, `get_index_stats()` method
- `src/ghps/indexer.py` — `dry_run` parameter, per-repo error handling, summary logging
- `src/ghps/search.py` — `indexed_at` and `freshness` fields on SearchResult, `_freshness_label()` function
- `tests/test_data_pipeline.py` — 14 new tests

## Decisions Made

- **Freshness labels use simple time buckets** (today/this_week/this_month/stale) rather than exact days — simpler UX and avoids timezone complexity.
- **Graceful 200 + error message** instead of 500 when no index exists — better for frontend error handling.
- **`_is_no_table_error()` uses type-name checking** because `pysqlite3.dbapi2.OperationalError` doesn't inherit from stdlib `sqlite3.OperationalError`.
