`★ Insight ─────────────────────────────────────`
**Key architectural decisions in this implementation:**

1. **Style injection from JS** — Since `style.css` was owned by another agent (multi-agent sprint constraint), CSS was injected via a self-executing function in `d3-viz.js`. This is a common pattern for encapsulated web components and avoids file ownership conflicts.

2. **viewBox-based responsiveness** — Instead of setting fixed `width`/`height` on the SVG, using `viewBox` with `width="100%"` lets the browser scale the entire visualization proportionally. This is the standard D3 approach for responsive SVGs and automatically handles the 375px mobile requirement.

3. **Safe DOM construction** — Tooltip content uses `createElement`/`createTextNode` rather than `innerHTML` to prevent XSS, even though the data comes from a static JSON file. Defense in depth matters for portfolio sites.
