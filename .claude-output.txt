`★ Insight ─────────────────────────────────────`
- **Defensive JSON parsing**: Instead of calling `.json()` directly (which throws on empty body), we first read `.text()`, check if it's non-empty, then `JSON.parse`. This prevents the confusing "Unexpected end of JSON input" error that was causing B-010.
- **Graceful degradation pattern**: The `useFallbackData()` function sets the app state to sample data *and* renders the UI, so the site still looks functional even when the backend has no data. The error banner is informational, not blocking.
